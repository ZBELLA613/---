2、需求分析
2.1 用户需求
角色	权限描述
系统管理员 	拥有全部管理权限，可进行用户管理、权限分配、数据维护、日志查看、报表导出 
普通教师 	只能查看和修改本人信息（包括本人基础信息、本人授课安排、本人科研成果） 
   2.2 功能需求

（1）教职工管理模块
 支持教职工基本信息的增删改查：工号、姓名、性别、出生日期、学历、学位、专业、入职时间、所属部门、职称、职务、薪资等级、联系方式
 支持教职工注销功能：标记状态为“在职/离职/退休/转出”，保留历史记录
 支持按姓名、编号、部门、职称、学历等多条件组合查询
 支持教职工信息表的打印及Excel导出

（2）课程管理模块
 支持课程信息的增删改查：课程编号、课程名称、课程性质（必修/选修/通识）、学分、课程时数、适用专业
 支持课程与授课教师的关联

（3）教师授课管理模块
 记录教师编号、课程编号、授课班级、学生人数
 支持按课程名称、授课教师查询教学任务安排
 支持教学任务表的打印及Excel导出

（4）科研项目管理模块
 支持课题信息的增删改查：课题编号、课题名称、立项单位、立项时间、结题时间、负责人
 支持按负责人、课题名称等条件查询

（5）专利管理模块
 支持专利信息的增删改查：专利编号、专利名称、授权时间、专利类型、发明人
 支持按发明人、专利类型等条件查询

（6）论文著作管理模块
 支持论文/著作信息的增删改查：作品名称、发表期刊/出版社、发表时间、作者排序、ISBN/ISSN号、第一作者
 支持按发表时间、作者等条件查询

（7）研究方向管理
 记录教师的研究方向信息

（8）奖惩记录管理模块
 支持奖惩记录的增删改查：奖惩类型（奖励/处罚）、原因、时间、发文编号
 与教职工信息关联

（9）科研成果统计与导出
 支持按教师、按时间范围统计科研成果
 支持科研成果表的打印及Excel导出

（10）安全审计功能
 基于角色的访问控制（RBAC）
 记录用户操作日志
 普通教师数据隔离（仅可见本人数据）

 2.3 数据需求

系统需要存储的核心数据实体包括：教职工、部门、课程、授课关系、科研项目、专利、论文著作、研究方向、奖惩记录、用户账号、操作日志等。

 2.4 性能需求
系统响应时间：常规查询操作应在3秒内完成
支持并发用户数：不少于20人
数据安全性：用户密码需加密存储，操作需记录审计日志
